wall velocity field

purpose

  • set velocity to wall particles for wall interactions

  • get wall velocity for bounce back on walls

data structures

data structures are separated into 2 kinds of data structures:

  • Wvel, which contains all informations needed for the velocity field. It is hold and maintained on host.

  • Wvel_v (Wvel view), containing only the information needed for one specific time. It is passed to device kernels.

interface

Interface is splitted between host and device

Host

Host code is responsible for initialisation of Wvel and convert Wvel to a view at a given timestep.

void ini(WvelCste p, Wvel *vw);
void ini(WvelShear p, Wvel *vw);
void ini(WvelShearSin p, Wvel *vw);
void ini(WvelHS p, Wvel *vw);

void step2params(long it, const Wvel *wv, /**/ Wvel_v *view);

Device

Device code contains the functions describing the field. Also contains bouncer function: gives bounce back velocity from position and velocity.

static __device__ void wvel(Wvel_v wv, Coords c, float3 r, /**/ float3 *v) {
    int type;
    wvel_fun wvel_funs[] = {&wvel_cste, &wvel_shear, &wvel_hs};
    type = wv.type;
    wvel_funs[type](wv.p, c, r, /**/ v);
}

static __device__ void bounce_vel(Wvel_v wv, Coords c, float3 rw, /* io */ float3* v) {
    float3 vw;
    wvel(wv, c, rw, /**/ &vw);
    v->x = 2 * vw.x - v->x;
    v->y = 2 * vw.y - v->y;
    v->z = 2 * vw.z - v->z;
}
